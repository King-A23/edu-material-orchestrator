# Security Policy

## Supported Versions

Security fixes are provided on a best-effort basis for the latest development snapshot and the most recent tagged release.

## Reporting a Vulnerability

Do not open a public issue for suspected vulnerabilities, leaked credentials, or accidentally committed restricted course material.

Report the issue privately through GitHub private vulnerability reporting for this
repository. If that flow is unavailable, contact a maintainer through an existing
trusted private channel before sharing details publicly.

Include:

- a short summary of the impact
- the affected command, file, or workflow
- reproduction details when safe to share
- whether the issue involves secrets, personal data, or restricted educational content

If the report involves exposed credentials, revoke or rotate them first when possible.

## Repository-Specific Risks

Please flag reports that involve any of these boundaries explicitly:

- copied proprietary skill content or prompts
- non-redistributable teaching materials, answer keys, or exam content
- student data, submissions, grades, or roster information
- external adapter workflows that may send sensitive content to third-party model providers
